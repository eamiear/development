# 享元模式： 一种用于性能优化的模式
> 核心是语用共享技术来有效支持大量的细粒度的对象


## 特点
  1. 内部状态
  2. 外部状态

### 状态划分依据
  1. 内部状态存储于对象内部
  2. 内部状态可以被一些对象共享
  3. 内部状态独立于具体场景，通常不会改变
  4. 外部状态取决于具体的场景，并根据场景二变化，外部状态不能被共享 

## 使用场景
  1. 系统中创建了大量类似的对象，而导致内存占用过高
  2. 由于使用了大量对象，造成很大的内存开销
  3. 对象的大多数状态都可以变为外部状态
  4. 剥离出对象的外部状态后，可以用相对较少的共享对象取代大量对象
